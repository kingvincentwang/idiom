Firebase 能做什麼？

你是否遇過以下需求？

  • 網站需要登入功能 (Google, FB)
  • 想儲存應用程式的動態資料
  • 想架設並發布網站到網際網路
  • 想傳送推播通知給使用者
  • 不想自己架設伺服器！

Firebase 就像一棟已經蓋好的科技大樓，你只要「租」裡面的服務即可，一個平台完成 App 的後端功能。

動手做：建立你的第一個 Firebase 專案

1

進入 Firebase Console

用 Google 帳號登入 console.firebase.google.com，點「建立專案」。

2

命名專案

輸入專案名稱（如 my-first-app），Google Analytics 可先停用，加快建立速度。

3

註冊 Web 應用程式

在專案總覽點 </> 圖示新增網頁應用程式，取得 firebaseConfig 設定碼。

4

啟用需要的服務

左側選單啟用 Authentication（選登入方式）與 Firestore Database（選測試模式、asia-east1）。

💡 firebaseConfig 就是你的專案「地址」——貼進網頁程式碼，前端就能直接連上雲端服務！

Authentication 身分驗證

生活化：大門警衛

進入門禁森嚴的校園：

  1. 訪客來到校門口
  2. 警衛查驗證件
  3. 發給專屬「通行證」
  4. 允許進入校園使用設施

Firebase 驗證

支援 Google, Apple, Email：

  1. 使用者來到你的網站
  2. Auth 處理第三方登入
  3. 核發專屬的「UID」
  4. 開始安全地存取資料庫

Firestore 雲端資料庫

比喻：Google 試算表

  • Collection (集合)：
    就像試算表的「分頁」
  • Document (文件)：
    就像表格中的「某一列」
  • Field (欄位)：
    具體的資料內容 (姓名、電話)

特性：NoSQL

  • 不需要預先定義資料表結構
  • 採用彈性的 JSON 樹狀結構
  • 即時同步： 一人更新，所有人畫面自動改變
  • 適合處理大規模數據擴展
前往申請免費的 Mistral API Key

Hosting 網站部署

發布網站到全世界

不需買虛擬主機，一行 firebase deploy 就能上線。

  • 免費 HTTPS： 自動配置 SSL
  • 全球 CDN： 快取在全球節點，載入極快
  • 快速回退： 一鍵還原到上個版本

Firebase 完整運作流程

各服務緊密相連，打造無伺服器 (Serverless) 體驗。

1. 驗證登入

透過 Auth 取得 UID。

2. 讀寫存取

憑證安全存取 Firestore。

3. 全球發布

網頁透過 Hosting 發布 CDN。

今日總結

服務 核心功能 比喻
Auth 第三方登入與身分驗證 大門警衛
Firestore 儲存即時動態資料 雲端試算表
Hosting 部署靜態網站與 CDN 網站空間

案例探討與練習

Auth 會員登入系統

  • 實作 Auth 會員登入系統（電子郵件/密碼註冊即登入）
  • 電子郵件/密碼註冊後須審核才能登入（加入 Firestore）
  • 使用 Google 帳號登入（不需註冊也不需審核）
  • 使用 Google 帳號登入，需註冊後等待審核

專案實作

  • 成語測驗系統（部署至 GitHub + Firebase）
  • 成語測驗系統 — 部署至 Firebase Hosting
  • 成語測驗系統：R-page + Firebase
  • R-page + GAS + Firebase 設計閱讀測驗
  • HTML + Firebase 試卷上傳系統（Google Chat 放前端或後端 GAS）
  • HTML + Firebase + Google Chat + Cloudflare（後端）試卷上傳即時通知
  • 純前端 HTML + Firestore + Auth（Google 登入審稿）+ Google Chat Webhook 通知
