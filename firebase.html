Firebase 入門教學

從零開始打造雲端應用

Google 提供的一站式 App 開發平台

Firebase 能做什麼？

你是否遇過以下需求？

  • 網站需要登入功能 (Google, FB)
  • 想儲存應用程式的動態資料
  • 想架設並發布網站到網際網路
  • 想傳送推播通知給使用者
  • 不想自己架設伺服器！

Firebase 就像一棟已經蓋好的科技大樓，你只要「租」裡面的服務即可，一個平台完成 App 的後端功能。

Firebase 核心架構

Firebase = BaaS (Backend as a Service)，免去管理 Server 的煩惱。

Auth

身分驗證

Firestore

即時資料庫

Storage

檔案儲存

Hosting

網站部署

Functions

雲端自動程式

Authentication 身分驗證

生活化：大門警衛

進入門禁森嚴的校園：

  1. 訪客來到校門口
  2. 警衛查驗證件
  3. 發給專屬「通行證」
  4. 允許進入校園使用設施

Firebase 驗證

支援 Google, Apple, Email：

  1. 使用者來到你的網站
  2. Auth 處理第三方登入
  3. 核發專屬的「UID」
  4. 開始安全地存取資料庫

Firestore 雲端資料庫

比喻：Google 試算表

  • Collection (集合)：
    就像試算表的「分頁」
  • Document (文件)：
    就像表格中的「某一列」
  • Field (欄位)：
    具體的資料內容 (姓名、電話)

特性：NoSQL

  • 不需要預先定義資料表結構
  • 採用彈性的 JSON 樹狀結構
  • 即時同步： 一人更新，所有人畫面自動改變
  • 適合處理大規模數據擴展

Cloud Storage 雲端硬碟

存放大型媒體檔案

📷 圖片、🎬 影片、📄 PDF 甚至是音樂，都可以存放在 Storage。

  • 使用者在前端上傳照片
  • 產生一組「下載網址」
  • 將該網址記錄到 Firestore

Storage 放檔案，Firestore 放資料！

Hosting 網站部署

發布網站到全世界

不需買虛擬主機，一行 firebase deploy 就能上線。

  • 免費 HTTPS： 自動配置 SSL
  • 全球 CDN： 快取在全球節點，載入極快
  • 快速回退： 一鍵還原到上個版本

Cloud Functions 雲端函式

比喻：待命的機器人

傳統伺服器一直開著很耗電：

  • 平常處於休息狀態 (不計費)
  • 「有人按門鈴」才開始工作
  • 把交代的工作快速做完
  • 做完馬上回去休息

用途：自動化腳本

設定「觸發條件」在雲端執行：

  • 資料庫： 新增留言時寄 Email
  • 驗證： 註冊時建立初始設定
  • 排程： 每天半夜統整報表

Firebase 完整運作流程

各服務緊密相連，打造無伺服器 (Serverless) 體驗。

1. 驗證登入

透過 Auth 取得 UID。

2. 讀寫存取

憑證安全存取 Firestore 與 Storage。

3. 背景觸發

資料變更觸發 Functions。

4. 全球發布

網頁透過 Hosting 發布 CDN。

今日總結

服務 核心功能 比喻
Auth 第三方登入與身分驗證 大門警衛
Firestore 儲存即時動態資料 雲端試算表
Storage 儲存大型媒體檔案 雲端硬碟
Hosting 部署靜態網站與 CDN 網站空間
Functions 雲端自動執行程式 機器人助手
